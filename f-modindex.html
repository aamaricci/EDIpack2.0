e
ed_aux_funx Hosts a number of auxiliary procedures required in different parts of the code. Specifically, it implements: creation/annihilation fermionic operators, binary decomposition of integer representation of Fock states and setup the local impurity Hamiltonian
ed_bath Contains routines for setting, accessing, manipulating and clearing the bath of the Impurity problem.
ed_bath_aux Implements a number of auxiliary procedures used to construct replica/general bath
ed_bath_dim Returns or check the dimensions to which the user should allocate the bath array.
ed_bath_dmft A class for the the :f:var:`effective_bath` data structure describing the effective bath in the code.
ed_bath_fit Contains routines that fit the Impurity model bath
ed_bath_functions A comprehensive set of procedures to evaluate the non-interacting impurity Green's functions :math:`\hat{G}^{\rm And}` and hybridizations :math:`\hat{F}^{\rm And}` in the complex frequency domain given the :f:var:`effective_bath` instance.
ed_bath_replica In these module we implement the functions to set the matrix basis :math:`\{ \hat{O}_i \}_{i=1,\dots,N_{sym}}` and the initial variational parameters :math:`\vec{\lambda}` used to decompose each local bath hamiltonian for the :f:var:`replica` and :f:var:`general` bath types.
ed_bath_user Implements functions the user can use to enforce specific symmetry operations on the bath array.
ed_chi_dens Evaluates the impurity density susceptibility.
ed_chi_exct Evaluates the impurity excitonc susceptibility.
ed_chi_pair Evaluates the impurity pair susceptibility.
ed_chi_spin Evaluates the impurity spin-spin susceptibility.
ed_diag
ed_diag_nonsu2
ed_diag_normal
ed_diag_superc
ed_eigenspace A class implementing a data structure to efficiently store the low part of the Fock space spectrum, automatically spreading and retrieving the eigenstates among/from MPI threads.
ed_gf_nonsu2
ed_gf_normal Constructs the interacting impurity electronic Green's functions. On request it evaluated the phononic Green's functions and and different susceptibilities.
ed_gf_superc
ed_greens_functions
ed_hamiltonian
ed_hamiltonian_nonsu2
ed_hamiltonian_nonsu2_direct_hxv
ed_hamiltonian_nonsu2_stored_hxv
ed_hamiltonian_normal Setup and build the sector Hamiltonian, returns the correct dimension of the vectors in the Arpack/Lanczos procedure in each thread and provides an interface to Tri-Diagonalize the Hamiltonian on a Krylov basis given a starting vector.
ed_hamiltonian_normal_common Global variables related to sector Hamiltonian construction. It contains the :f:func:`vector_transpose_mpi` implementing the MPI :code:`Allv-2-Allv` parallel matrix transposition.
ed_hamiltonian_normal_direct_hxv Constructs and applies on-the-fly each term of the sector Hamiltonian to the input vector :math:`\vec{w} = H\times \vec{v}` in a Arpack/Lanczos framework.
ed_hamiltonian_normal_stored_hxv Constructs each terms of the sector Hamiltonian storing them into different :f:var:`sparse_matrix` instances, implement the corresponding matrix-vector products using stored sparse matrices.
ed_hamiltonian_superc
ed_hamiltonian_superc_direct_hxv
ed_hamiltonian_superc_stored_hxv
ed_input_vars Contains all global input variables which can be set by the user through the input file. A specific preocedure :f:func:`ed_read_input` should be called to read the input file using :f:func:`parse_input_variable` procedure from SciFortran. All variables are automatically set to a default, looked for and updated by reading into the file and, sequentially looked for and updated from command line (std.input) using the notation `variable_name=variable_value(s)` (case independent).
ed_io Contains a set of routines that retrieve quantities such as Green's functions, self-energies (see :f:mod:`ed_greens_functions` ) and observables (from :f:mod:`ed_observables` ) and pass them to the user, as well ass routines to read and store Green's function and self-energies.
ed_main Contains routine that initialize, run and finalize the Impurity model solver
ed_observables
ed_observables_nonsu2 This module calculates a series of observables, and stores them in aptly named plain-text files. :f:var:`ed_mode` = :code:`nonsu2`
ed_observables_normal This module calculates a series of observables, and stores them in aptly named plain-text files. :f:var:`ed_mode` = :code:`normal`
ed_observables_superc
ed_rdm
ed_rdm_nonsu2
ed_rdm_normal
ed_rdm_superc
ed_sector Contains procedures to construct the symmetry sectors corresponding to a given set of quantum numbers :math:`\vec{Q}`, in particular it allocated and build the :f:var:`sector_map` connecting the states of a given sector with the corresponding Fock ones.
ed_setup Contains procedures to set up the Exact Diagonalization calculation, executing all internal consistency checks and allocation of the global memory.
ed_sparse_matrix This class defines a data structure to efficiently store sparse matrices into dedicated CSR matrices, featuring support to MPI parallel storage, so that each matrix is spread across the threads.
ed_vars_global Contains all variables, arrays and derived types instances shared throughout the code. Specifically, it contains definitions of the :f:var:`effective_bath`, the :f:var:`gfmatrix` and the :f:var:`sector` data structures.
